While our team originally intended for our game to be a web-browser game, HTML5 uploads from GameMaker have been notoriously buggy. As a result, we offer two different deployment strategies: as a web-browser playable game and as a downloadable executable. 

Our deployment environment of choice was Itch.io, a popular hosting platform for indie game developers, due to its ease of convenience. 

Please note that in order to deploy games in GameMaker, an OperaGX account will need to be made. 

### Browser Playable Game

In GameMaker, set your target build to HTML5 (Click on the target icon in the top right corner). Create an executable, define your game options (i.e. browser title, splash screen, etc.) and choose to package it as a ZIP file. 
From there, go to the Itch.io website, make an account, and upload your zipped file. Fill out any relevant details on the upload page. 

Your game should now be playable in the browser!

### Downloadable Executable

Set your target build to the OS you want to target (ex. Windows). Create an executable, but this time, package it as an installer. (An installer will be a single executable that installs your entire game while the ZIP file will need to be downloaded and extracted in order for the game to run.) 
Go to the Itch.io website and follow the same steps as if you were uploading a browser playable game, but mark it as an downloadable rather than an HTML project. Fill out any relevant details. 

Your game should now be playable as an executable!
